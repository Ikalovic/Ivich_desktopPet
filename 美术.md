# 美术资源生产规范

这份文档用于统一 Q 版桌宠的角色设计、一致性控制和 GPT Image 出图流程，目标是让 `assets/` 下的资源尽量保持为同一个角色，便于后续动画、程序接入和资源迭代。

## 目标

- 保证不同动作、表情、状态下始终是同一个角色
- 降低模型在多轮生成中出现的人设漂移
- 让美术资源命名、尺寸、提示词和出图流程可复用

## 核心原则

不要把每一张图都当成一次全新生成。

正确方式是：

1. 先做一张角色母版图
2. 固化角色设定
3. 后续所有图片尽量基于母版图或上一帧做编辑
4. 每次只修改动作、表情或姿势，不重做人设

这比每次“重新生成一个类似角色”稳定得多。

## 为什么会漂移

即使使用同一套提示词，纯重新生成仍然可能出现这些问题：

- 发型长度变化
- 发夹、饰品消失或换位置
- 眼睛大小和脸型变化
- 服装细节和颜色偏移
- 头身比不稳定

所以项目流程必须优先使用“参考图 + 编辑”，而不是“只靠文字重画”。

## 推荐生产流程

### 1. 先锁定角色母版

第一阶段不要急着做动画，先做角色基准图：

- `base_front.png`
- `base_side.png`
- `base_three_quarter.png`
- `expression_sheet.png`

建议这些基准图先放在：

- `assets/temp/`

等角色稳定后，再拆分进入正式目录。

### 2. 建立角色圣经

角色圣经是固定不变的人设描述，后续每次出图都要重复使用。

建议至少包含：

- 角色名
- 头身比
- 脸型
- 发型
- 发色
- 瞳色
- 服装
- 标志性饰品
- 画风
- 线稿风格
- 上色风格
- 背景要求

### 3. 所有动作图用编辑模式生产

推荐顺序：

1. 用 GPT Image 生成角色母版图
2. 选出最稳定的一张作为唯一基准
3. 后续所有动作图都基于该图编辑
4. 动画帧优先基于上一帧继续编辑

例如：

- `idle_01` 基于母版图编辑
- `idle_02` 基于 `idle_01` 编辑
- `idle_03` 基于 `idle_02` 编辑

同理：

- `walk_01` 基于侧面或 3/4 视角基准图编辑
- `walk_02` 基于 `walk_01` 编辑
- `walk_03` 基于 `walk_02` 编辑

### 4. 先做关键帧，再补中间帧

不要一开始就生成整套 8 帧或 12 帧。

先做这些关键节点：

- 待机主姿势
- 行走抬左脚
- 行走重心中间
- 行走抬右脚
- 睡觉主姿势
- 拖拽主姿势
- 摔倒起始和结束

关键帧稳定后，再补帧。

### 5. 情绪尽量独立制作

如果程序方案允许，建议把表情做成单独资源：

- `happy.png`
- `angry.png`
- `cry.png`
- `shy.png`
- `surprise.png`

这样可以降低重复重画整个人体带来的人设漂移。

## 本项目建议统一规范

### 画布规范

- 尺寸：`512 x 512`
- 背景：透明
- 锚点：底部居中
- 角色占画面高度：约 `70%`
- 角色默认朝向：优先朝右
- 光照方向：左上

### 角色绘制规范

- 风格：日系 Q 版
- 头身比：`2.5` 头身左右
- 线条：粗描边，边缘清晰
- 上色：柔和赛璐璐上色
- 明暗：不过度写实
- 轮廓：适合抠图和透明背景展示

### 文件命名规范

- 序列帧统一两位数：`idle_01.png`
- 特效序列帧统一两位数：`heart_01.png`
- 单张表情资源：`happy.png`
- 音频可使用：`click_01.wav`

## 通用提示词结构

每次出图都建议拆成 4 段：

1. 固定角色描述
2. 本次动作或表情要求
3. 输出规格
4. 禁止项

结构如下：

```text
[固定角色描述]
[本次动作/表情要求]
[输出规格]
[禁止项]
```

## 特效资源建议

特效可以比角色自由一些，但建议仍统一视觉风格：

- 描边风格与角色接近
- 饱和度略高于角色
- 透明背景
- 单个特效不要过大

建议目录对应如下：

- `assets/effects/heart/`
- `assets/effects/star/`
- `assets/effects/smoke/`
- `assets/effects/zzz/`

示例要求：

- `heart`: 爱心飘出，软萌
- `star`: 小星星闪烁
- `smoke`: 碰撞或消失时的小烟团
- `zzz`: 睡眠状态漂浮符号

## 出图顺序建议

建议按下面顺序推进，而不是全部一起做：

1. 角色母版
2. 正侧面基准图
3. `idle_01`
4. `idle_02`
5. `walk_01`
6. `walk_02`
7. `sleep_01`
8. `drag_01`
9. `fall_01`
10. 表情资源
11. 特效资源

这样可以先锁定角色，再扩展动作。

## 验收检查表

每生成一张图，至少检查下面几点：

- 是不是还能一眼认出是同一个角色
- 发型长度是否一致
- 发夹是否还在同一位置
- 服装颜色是否漂移
- 脸型和五官是否明显变化
- 头身比是否失真
- 透明背景是否干净
- 轮廓是否适合后续逐帧动画

如果有明显漂移，不建议继续在错误结果上迭代，应回退到最近稳定版本重新编辑。
