# 下一阶段出图计划

你已经有以下 4 张基准图：

- `base_front.png`
- `base_side.png`
- `base_three_quarter.png`
- `expression_sheet.png`

下一步不要急着铺满全部资源，先生成一组“动作定标图”。目标是先确认这个角色在不同状态下仍然稳定，再决定是否补中间帧。

## 推荐顺序

建议按下面顺序生成：

1. `assets/character/idle/idle_01.png`
2. `assets/character/idle/idle_02.png`
3. `assets/character/walk/walk_01.png`
4. `assets/character/walk/walk_02.png`
5. `assets/character/sleep/sleep_01.png`
6. `assets/character/drag/drag_01.png`
7. `assets/character/fall/fall_01.png`
8. `assets/character/emotion/happy.png`

原因：

- `idle_01` 是后续大多数动作的稳定起点
- `walk_01` 和 `walk_02` 可以先验证侧面动作时的人设一致性
- `sleep_01` 可以验证头发、龙角、龙翼在收拢姿态下是否稳定
- `drag_01` 和 `fall_01` 可以验证大幅姿态变化时会不会崩
- `happy.png` 用来验证表情变化时脸部是否还能保持同一角色

## 固定角色设定

下面这段在后续所有 prompt 里都应保持不变：

```text

2.5头身，圆脸，大眼睛，小嘴，白色及腰长发，挑染天蓝色，

白蓝金配色的幻想系圣女服装，高领披肩，宽大袖口，短裙，多层裙摆，
发光的星月纹路，袜子带金色花纹，整体对称设计，治愈感，神圣感，三无，
JRPG幻想风，桌宠角色设计。
可爱，清爽，粗描边，柔和赛璐璐上色，轮廓清晰，浅绿色背景。
```

## 出图规则

- 优先使用“编辑输入图片”
- `idle_02` 基于 `idle_01` 编辑，不要重新从 `base_front.png` 开始
- `walk_02` 基于 `walk_01` 编辑，不要重新从 `base_side.png` 开始
- 每次只改一个变量：动作、表情或姿势
- 统一要求：单人物、全身、透明背景、角色居中、四周留边

## 质量判断标准

只要出现下面任一问题，就不要继续在该结果上补帧，应回退到上一张稳定图：

- 月亮发夹位置变了
- 龙角渐变方向变了
- 龙翼形状不一致
- 头发长度明显变短或变厚
- 服装层数、袖口、高领披肩结构发生变化
- 白蓝金主配色漂移
- 脸型或眼睛比例明显变化
